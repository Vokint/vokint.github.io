Chapter 2 - Computation Complexity (Big-O)

There are two types of complexity

  • Time Complexity (Time spent)
  • Space Complexity (Memory space spent)

They both use three asymptotic notation to descript the complexity

  • $ \textnormal{Big-O (O)} $
  • $ \textnormal{Big-Omega }(\Omega) $
  • $ \textnormal{Big-Theta }(\Theta) $

Big-O

Upper bound of the complexity

f(n)=O(g(n))f(n) = \textnormal{O}(g(n)) f(n)=O(g(n))

Big-Omega

Lower bound of the complexity

f(n)=Ω(g(n))f(n) = \Omega(g(n)) f(n)=Ω(g(n))

Big-Theta

Exact bound of the complexity

f(n)=Θ(g(n))f(n) = \Theta(g(n)) f(n)=Θ(g(n))

Common Complexity Classes

  • Constant time - $ O(1) $
  • Logarithmic time - $ O(\log_2 n) $
  • Linear time - $ O(n) $
  • Log-linear time - $ O(n\log_2 n) $
  • Quadratic time - $ O(n^2) $
  • Cubic time - $ O(n^3) $
  • Polynomial time - $ O(n^k) $
  • Exponential time - $ O(2^n) $
  • Factorial time - $ O(n!) $

Slope of Complexity Classes

TIme Complexity

Simple operations such as

  • declaration, assignment
  • cin, cout
  • function call
  • return

take constant amount of time
The overall time complexity = O(Constant),
and constant time costs almost zero influence no matter the size.
Therefore the overall time complexity is

O(1)O(1) O(1)

Only looping will cost longer time complexity
Single nested loop usually takes constant time * size of loop as the operations C inside the loop will run n times.

C∗nC * n C∗n

The overall time complexity is

O(n)O(n) O(n)

While for more than one nested loop usually takes constant time * (n ^ depth of nest) as the operations C inside the loop will run n * n * ... times.

C∗n2C * n^2 C∗n2

The overall time complexity is

O(n2)O(n^2) O(n2)

Introduction of Hashing

Hashing is generally [ Indexing ].

If you have learnt what [ index ] is, you may understand it as positioning or classifying objects with duplicated meaning for a smaller size of data and better searching.

Steps for Indexing

  1. Define an array with a specific purpose.
  2. Define meaning for each index in the array.
  3. Classify the input and determine the size of the array.
  4. Map the input to the array.

Example of Indexing (Minimal Hash)

Given A string [foo] = “abacbdef”.
Precondition: there will only be alphabets inside the string.
If you want to find how many ‘a’ occured inside the string, you may use Indexing.

  1. Define an array for storing the occurrence of letters.
  2. Each index represents an alphebet.
  3. There are a total of 26 alphabets as potential input, so the size of array will be 26.
  4. index[0] represents the occurrence of ‘a’, index[1] represents the occurrence of ‘b’, and so on…
  5. Loop over the string and map the character to corresponding index.

However, there are still some differences between Hashing and Indexing.
In Hashing, we convert ASCII char to key directly with foo[i] - 'a' so that ‘a’ = 0, ‘b’ = 1, and so on…
which allows us to map the data easier and faster.

Hash function

A hash function is a little bit more than just indexing.
It converts a large possibly variable-sized amount of data into a small datum,
but with various mapping methods and transforms data into keys directly.

Example of Hashing

Hashing Student ID k using Modulo, where h() is the function and h(k) is the hash value of k.

h(k)=k mod nh(k) = k \bmod n h(k)=kmodn

The size of array will reduce a lot due to the range of modulo [0 to n-1].
Take k= 5000, n = 5,

h(k)=k mod 5h(k) = k \bmod 5 h(k)=kmod5

k -> arr[h(k)]

h(5000)=5000 mod 5=0h(5000) = 5000 \bmod 5 = 0 h(5000)=5000mod5=0

5000 -> arr[0], 5081 -> arr[1], 5072 -> arr[2], 5023 -> arr[3], 5014 -> arr[4]

A hash value is usually an integer that may serve as an index to an array.

Perfect Hash

A hash function that maps each valid input to a different hash value.

Minimal Hash

A hash funciton that maps n input keys to n consecutive hash values.

Hash Collision

Collision occurs when two distinct data own the same hash value.
Collisions are usually unavoidable when a large set of data is mapped into small datum.
They will increase the average cost of lookup operations.

Solutions to Hash Collision

  • Opening Addressing
  • Closing Addressing

Opening Addressing

Ideal properties

  • Low Cost
    • Easier and faster to compute the result
  • Variable Range

  • Uniformity
    • Distribute the keys evenly
    • Minimize the chance of collisions
